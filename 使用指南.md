# 刘欢の博客 · 本地使用指南

本博客基于 [Astro](https://astro.build) + [Fuwari](https://github.com/saicaca/fuwari) 主题，从原来的 Hexo 迁移而来。
这份文档教你**一步步在本地把博客跑起来、写文章、改配置、构建部署**。所有命令都在 `blog-astro` 目录下执行。

---

## 一、环境准备（只需做一次）

### 1. 安装 Node.js

需要 **Node.js ≥ 20**。检查版本：

```powershell
node -v
```

如果没有或版本太低，去 <https://nodejs.org> 下载 LTS 版本安装。

### 2. 安装 pnpm（本项目只能用 pnpm）

需要 **pnpm ≥ 9**。检查：

```powershell
pnpm -v
```

如果没有，用 Node 自带的 corepack 启用（推荐）：

```powershell
corepack enable
corepack prepare pnpm@latest --activate
```

> ⚠️ 本项目锁定了 pnpm，用 `npm install` 或 `yarn` 会被拦截报错，请务必用 `pnpm`。

### 3. 安装项目依赖

进入项目目录后执行（首次会下载较多内容，耐心等待）：

```powershell
pnpm install
```

## 二、本地启动（日常预览）

```powershell
pnpm dev
```

启动后终端会显示一个地址，通常是 <http://localhost:4321>。
用浏览器打开即可看到博客，**改完文件会自动热更新**，不用重启。

按 `Ctrl + C` 停止。

## 三、写一篇新文章

### 方式 A：用脚本自动生成（推荐）

```powershell
pnpm new-post 我的新文章
```

这会在 `src/content/posts/` 下生成 `我的新文章.md`，并自动填好日期等头部信息。
文件名建议用英文/拼音（会成为网址的一部分），例如：

```powershell
pnpm new-post my-first-post
```

### 方式 B：手动复制一个已有的 `.md` 改

文章都在 `src/content/posts/` 目录里，复制一篇改内容即可。

### 文章头部信息（frontmatter）说明

每篇文章开头 `---` 之间的内容是元信息：

```markdown
---
title: "文章标题"              # 标题
published: 2024-06-02         # 发布日期
description: "一句话简介"      # 列表页显示的摘要
image: ""                    # 头图，留空则不显示；可填 /img/header_img/xxx.jpg
tags: ["标签1", "标签2"]      # 标签
category: "笔记"              # 分类
draft: false                 # true=草稿不发布，false=正式发布
---
```

写完正文保存，回到浏览器就能看到。

### 文章里怎么插入图片

把图片放到 `public/` 目录下，然后在正文里用 `/` 开头的路径引用：

- 头图：放 `public/img/header_img/`，frontmatter 里写 `image: "/img/header_img/图片名.jpg"`

- 正文图：放 `public/cdn/`（已有 `life`、`note` 子目录），正文里写：
  
  ```markdown
  ![描述](/cdn/life/图片名.jpg)
  ```

> 记住规律：**放进 `public/` 的文件，引用时把 `public` 去掉、用 `/` 开头**。

## 四、常见自定义

所有站点配置都在 **`src/config.ts`**，常改的几处：

| 想改什么        | 改哪里                                     |
| ----------- | --------------------------------------- |
| 网站标题 / 副标题  | `siteConfig.title` / `subtitle`         |
| 主题色         | `siteConfig.themeColor.hue`（0–360 的色相值） |
| 头像          | 替换 `src/assets/images/avatar.jpg`       |
| 个人简介 / 社交链接 | `profileConfig`                         |
| 导航栏菜单       | `navBarConfig.links`                    |
| 关于页内容       | `src/content/spec/about.md`             |

### 换网站图标（favicon）

当前图标是用头像生成的。如果换了头像想同步更新图标，执行：

```powershell
node scripts/gen-favicon.mjs
```

它会读取 `src/assets/images/avatar.jpg`，重新生成 `public/favicon/favicon-avatar-*.png`（已在 `config.ts` 中接好，无需再改配置）。

### 改歌单（音乐页）

音乐页在 **`src/pages/music.astro`**，顶部有两个数组：

- `neteasePlaylists`：网易云歌单，填**歌单 ID**即可（歌单网址里 `id=` 后面的数字）
- `qqPlaylists`：QQ 音乐歌单，填标题和完整网址

> 网易云用官方播放器直接嵌入；QQ 音乐没有稳定的网页内嵌播放器，所以做成点击跳转的卡片。

## 六、改完内容后，怎么上线（日常最常用）

**核心结论先记住：**

- ✅ **你不需要在本地 `pnpm build`。** 只要把改动推到 GitHub 的 `main` 分支，Vercel 会自动帮你构建（它在云端跑 `pnpm build`）并上线。
- ✅ 本地的 `pnpm build` / `pnpm preview` 只是**自己想提前预览成品**时才用，不是上线的必要步骤。
- ✅ 你改了哪些文件不用记，git 会自动帮你检测。

下面是**从"我改完了"到"网站更新了"的完整一步步流程**。所有命令都在 `blog-astro` 目录下执行。

### 第 1 步：确认你改的东西在正确的位置

| 你改了什么 | 文件在哪 |
| ---- | ---- |
| 写/改文章 | `src/content/posts/` 下的 `.md` 文件 |
| 文章配图、头图 | `public/` 目录（如 `public/cdn/`、`public/img/header_img/`） |
| 站点配置（标题、菜单、简介等） | `src/config.ts` |
| 关于页 | `src/content/spec/about.md` |
| 歌单 | `src/pages/music.astro` |

> 只要是项目目录里的文件，git 都能检测到，不用专门记路径。

### 第 2 步（可选）：本地预览确认没问题

不放心的话，先本地看一眼：

```powershell
pnpm dev
```

浏览器打开 <http://localhost:4321> 检查效果，没问题按 `Ctrl + C` 停止。
（这一步可跳过，但建议改完最好看一眼再上传。）

### 第 3 步：看看自己改动了哪些文件

```powershell
git status
```

红色的是有改动/新增的文件。确认这些就是你想上传的内容。

### 第 4 步：把所有改动加入暂存区

```powershell
git add .
```

> `.` 表示"当前目录所有改动"。如果只想传某个文件，可以写 `git add src/content/posts/我的文章.md`。

### 第 5 步：提交，并写一句说明

```powershell
git commit -m "新增文章：我的新文章"
```

引号里随便写你这次干了啥，方便以后回看，例如 `"修改关于页"`、`"更新歌单"`。

### 第 6 步：推送到 GitHub

```powershell
git push
```

> 第一次推送如果提示要登录，按提示用 GitHub 账号授权一次即可，以后就不用再登了。
> 当前仓库：`https://github.com/147cai/blog.git`，分支 `main`。

### 第 7 步：等待 Vercel 自动上线

`git push` 成功后**什么都不用做**：

1. Vercel 检测到 `main` 分支有新提交，自动开始构建。
2. 大约 **1～3 分钟**后部署完成，网站 <https://liuhuanblog.top/> 就更新了。
3. 想看构建进度，可登录 <https://vercel.com>，进入这个项目的 **Deployments** 页面查看。

如果网页没更新，多半是浏览器缓存，按 `Ctrl + F5` 强制刷新一下。

---

### 一句话流程（熟练后照着敲就行）

```powershell
git add .
git commit -m "这次改了啥"
git push
```

推完等 1～3 分钟，网站自动更新，**不用本地构建**。

---

### 附：首次配置 Vercel（已配好，了解即可）

如果将来要换一个新仓库或重新配置，步骤是：

1. 把 `blog-astro` 目录初始化为 git 仓库并推到 GitHub。
2. 在 <https://vercel.com> 用 GitHub 账号登录，导入该仓库。
3. Vercel 会自动识别 Astro，构建命令用 `pnpm build`、输出目录 `dist`（项目已自带 `vercel.json`）。
4. 把 `astro.config.mjs` 里的 `site` 改成正式域名，再提交一次。

> 本博客已经配好，日常只需照上面"第 1～7 步"操作即可。
